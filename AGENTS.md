# PÁTINA Catalogue Project Instructions

## Brand Principles
- Treat PÁTINA as a premium handcrafted glassware brand with an artistic, calm, and editorial point of view.
- Keep the catalogue feeling curated, intentional, and refined rather than commercial or loud.
- Preserve authenticity. Present each piece as a handcrafted object, not generic inventory.
- Never suggest changing the painted artwork unless the user explicitly asks.
- Default customer-facing language to concise, tasteful Spanish unless the surrounding surface clearly uses another language.

## Visual Principles
- Preserve the current design direction unless there is a strong reason to change it.
- Prefer elegant minimalism: clear hierarchy, generous breathing room, restrained motion, and deliberate composition.
- Use a few strong visual decisions instead of many decorative ones.
- Keep typography, spacing, borders, shadows, and gradients soft and controlled.
- Avoid visual clutter, badge-heavy layouts, crowded grids, and generic ecommerce patterns.
- Let product imagery and editorial composition do most of the storytelling.

## Copy Principles
- Keep copy concise, intentional, and understated.
- Prefer short headings, short supporting lines, and restrained microcopy.
- Avoid generic ecommerce language, hype, urgency, and exaggerated luxury claims.
- Do not over-explain craftsmanship where the photography and layout already communicate it.
- Use language that feels composed, tactile, and premium.

## Image Authenticity Rules
- Preserve the real glass shape, proportions, painted detail, and color fidelity.
- Do not crop, zoom, or frame images in a way that distorts the cup or hides important artwork.
- Keep product images, context images, detail shots, and process images consistent with the site's editorial tone.
- Avoid overlays or treatments that compete with the object.
- Use factual, restrained alt text that describes the object or scene clearly.

## Responsiveness Expectations
- Design for desktop and mobile together, not as separate afterthoughts.
- Keep layouts balanced at mobile, tablet, and desktop widths.
- Preserve calm hierarchy on small screens: readable line lengths, stable image ratios, and clear CTA placement.
- Avoid mobile layouts that feel cramped and desktop layouts that feel empty without purpose.

## Safe Editing Behavior
- Inspect the relevant files before changing them and align with existing repo patterns.
- Prefer focused edits over broad rewrites.
- Do not revert user changes or unrelated local modifications.
- Do not expose, commit, or duplicate secrets.
- Preserve product truth. Do not invent product details, collections, pricing meaning, or artistic techniques.
- Respect the current project structure and add helper files only when they clearly improve reuse.

## Validation Expectations
- After UI or layout edits, review hierarchy, spacing, image balance, and mobile/desktop behavior.
- After copy edits, review tone, brevity, and consistency with nearby sections.
- After code edits, run `npm.cmd run lint` when feasible.
- After structural changes that affect rendering or build output, run `npm.cmd run build` when feasible.
- Call out unresolved warnings, assumptions, and anything that still needs manual verification.
